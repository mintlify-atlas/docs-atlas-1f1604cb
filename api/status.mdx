## Overview

`Status` represents an HTTP status code along with its category and reason phrase. It provides type-safe access to standard HTTP status codes and supports custom status codes.

```scala
sealed trait Status extends Product with Serializable
```

## Status Categories

Status codes are organized into sealed trait hierarchies:

```scala
sealed trait Informational extends Status  // 100 – 199
sealed trait Success       extends Status  // 200 – 299
sealed trait Redirection   extends Status  // 300 – 399
sealed trait Error         extends Status  // 400 – 599
sealed trait ClientError   extends Error   // 400 – 499
sealed trait ServerError   extends Error   // 500 – 599
```

## Custom Status Codes

```scala
final case class Custom(override val code: Int, override val reasonPhrase: String = "") extends Status
```

<ParamField path="code" type="Int" required>
  The numeric status code
</ParamField>

<ParamField path="reasonPhrase" type="String" default="">
  Custom reason phrase
</ParamField>

```scala
// Custom status code
val customStatus = Status.Custom(299, "Custom Success")
customStatus.code // 299
customStatus.reasonPhrase // "Custom Success"

// Custom status without reason
val minimal = Status.Custom(450)
minimal.code // 450
minimal.reasonPhrase // ""
```

## Creating Status from Values

### From Int

```scala
def fromInt(code: Int): Status
```

<ParamField path="code" type="Int" required>
  The status code number
</ParamField>

<ResponseField name="result" type="Status">
  Returns the corresponding standard status or a `Custom` status if not recognized
</ResponseField>

```scala
val status = Status.fromInt(200)  // Status.Ok
val notFound = Status.fromInt(404) // Status.NotFound
val custom = Status.fromInt(999)   // Status.Custom(999, "")
```

### From String

```scala
def fromString(code: String): Option[Status]
```

<ParamField path="code" type="String" required>
  String representation of status code
</ParamField>

<ResponseField name="result" type="Option[Status]">
  Returns `Some(status)` if the string is a valid integer, `None` otherwise
</ResponseField>

```scala
val status = Status.fromString("200")    // Some(Status.Ok)
val invalid = Status.fromString("abc")   // None
val custom = Status.fromString("999")    // Some(Status.Custom(999, ""))
```

## Type Signature

```scala
package zio.http

import zio.Trace

sealed trait Status extends Product with Serializable {
  val code: Int
  val reasonPhrase: String
  lazy val text: String
  
  def isInformational: Boolean
  def isSuccess: Boolean
  def isRedirection: Boolean
  def isClientError: Boolean
  def isServerError: Boolean
  def isError: Boolean
  
  def toRoutes(implicit trace: Trace): Routes[Any, Nothing]
  def toResponse: Response
}

object Status {
  sealed trait Informational extends Status
  sealed trait Success       extends Status
  sealed trait Redirection   extends Status
  sealed trait Error         extends Status
  sealed trait ClientError   extends Error
  sealed trait ServerError   extends Error
  
  final case class Custom(override val code: Int, override val reasonPhrase: String = "") extends Status
  
  def fromInt(code: Int): Status
  def fromString(code: String): Option[Status]
  
  // Standard status codes (case objects)
  case object Ok extends Success { override val code: Int = 200 }
  case object NotFound extends ClientError { override val code: Int = 404 }
  // ... (all other standard statuses)
}
```

## Examples

### Basic Response Creation

```scala
import zio.http._

// Simple responses
val okResponse = Response.status(Status.Ok)
val notFoundResponse = Response.status(Status.NotFound)

// With body
val jsonResponse = Response(
  status = Status.Ok,
  body = Body.fromString("""{ "message": "Success" }""")
    .contentType(MediaType.application.json)
)
```

### Status-Based Routing

```scala
import zio.http._

val routes = Routes(
  Method.GET / "health" -> Handler.ok,
  Method.GET / "not-implemented" -> Handler.status(Status.NotImplemented)
)
```

### Conditional Status

```scala
import zio.http._

def handleRequest(isValid: Boolean): Response = {
  val status = if (isValid) Status.Ok else Status.BadRequest
  Response.status(status)
}

// Using pattern matching
def handleResult[A](result: Either[String, A]): Response = result match {
  case Right(value) => Response(
    status = Status.Ok,
    body = Body.fromString(value.toString)
  )
  case Left(error) => Response(
    status = Status.BadRequest,
    body = Body.fromString(error)
  )
}
```

### Error Handling

```scala
import zio.http._

val handler = Handler.fromFunctionZIO[Request] { request =>
  processRequest(request)
    .map(result => Response(
      status = Status.Ok,
      body = Body.fromString(result)
    ))
    .catchAll {
      case _: IllegalArgumentException => 
        ZIO.succeed(Response.status(Status.BadRequest))
      case _: NoSuchElementException =>
        ZIO.succeed(Response.status(Status.NotFound))
      case _ =>
        ZIO.succeed(Response.status(Status.InternalServerError))
    }
}
```

### Status Code Inspection

```scala
import zio.http._

def logResponse(response: Response): Unit = {
  val status = response.status
  
  if (status.isSuccess) {
    println(s"Success: ${status.code} ${status.reasonPhrase}")
  } else if (status.isClientError) {
    println(s"Client error: ${status.code} ${status.reasonPhrase}")
  } else if (status.isServerError) {
    println(s"Server error: ${status.code} ${status.reasonPhrase}")
  }
}

// Filtering by status
val responses: List[Response] = ???
val errors = responses.filter(_.status.isError)
val successes = responses.filter(_.status.isSuccess)
```
