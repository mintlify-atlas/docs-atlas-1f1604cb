`Route` represents a single HTTP route with a specific pattern and handler. Routes can either handle their errors (converting them to responses) or have polymorphic errors that must be handled before execution.

## Type Signature

```scala
sealed trait Route[-Env, +Err]
```

<ParamField path="Env" type="type parameter">
  The environment type required by the route's handler
</ParamField>

<ParamField path="Err" type="type parameter">
  The error type that can be produced by the route
</ParamField>

## Creating Routes

### Route.route

```scala
def route[Params, Env](rpm: RoutePattern[Params]): UnhandledConstructor[Env, Params]
```

Creates a route from a route pattern. Returns a constructor that accepts a handler.

<ParamField path="rpm" type="RoutePattern[Params]">
  The route pattern specifying method and path
</ParamField>

**Example:**

```scala
import zio.http._

Route.route(Method.GET / "users" / int("id")) { (id: Int, request: Request) =>
  Handler.text(s"User ID: $id")
}
```

### Route.handled

```scala
def handled[Params, Env](rpm: RoutePattern[Params]): HandledConstructor[Env, Params]
```

Creates a route that has already handled its errors (producing only responses).

### Route.notFound

```scala
val notFound: Route[Any, Nothing]
```

A route that always returns 404 Not Found.

## Error Handling

### handleError

```scala
def handleError(f: Err => Response)(implicit trace: Trace): Route[Env, Nothing]
```

Handles all typed errors by converting them into HTTP responses.

<ParamField path="f" type="Err => Response">
  Function to convert errors into responses
</ParamField>

<ResponseField name="return" type="Route[Env, Nothing]">
  A route that cannot fail (all errors converted to responses)
</ResponseField>

**Example:**

```scala
route.handleError { 
  case e: NotFoundException => Response.notFound(e.getMessage)
  case e: ForbiddenException => Response.forbidden(e.getMessage)
}
```

### handleErrorZIO

```scala
def handleErrorZIO[Env1 <: Env](f: Err => ZIO[Env1, Nothing, Response])(implicit trace: Trace): Route[Env1, Nothing]
```

Handles errors effectfully, allowing access to ZIO environment.

<ParamField path="f" type="Err => ZIO[Env1, Nothing, Response]">
  Effectful function to convert errors into responses
</ParamField>

### handleErrorCause

```scala
def handleErrorCause(f: Cause[Err] => Response)(implicit trace: Trace): Route[Env, Nothing]
```

Handles both typed errors and defects by examining the full `Cause`.

<ParamField path="f" type="Cause[Err] => Response">

### handleErrorCauseZIO


### handleErrorRequest

```scala
def handleErrorRequest(f: (Err, Request) => Response)(implicit trace: Trace): Route[Env, Nothing]
```

Handles errors with access to the request that caused the error.

<ParamField path="f" type="(Err, Request) => Response">
  Function receiving both error and request
</ParamField>

**Example:**

```scala
route.handleErrorRequest { (err, req) =>
  Response.internalServerError(s"Error processing ${req.path}: ${err.getMessage}")
}
```

### handleErrorRequestCause


## Path Manipulation

### nest

```scala
def nest(prefix: PathCodec[Unit]): Route[Env, Err]
```

Nests the route under a path prefix.

<ParamField path="prefix" type="PathCodec[Unit]">
  The path prefix to add
</ParamField>

**Example:**

```scala
val userRoute = Route.route(Method.GET / "profile") { ... }
val nestedRoute = userRoute.nest(Root / "api" / "v1")
// Now matches: GET /api/v1/profile
```

## Internal Variants

The `Route` sealed trait has four internal implementations:

- **`Handled`**: Route with all errors handled (produces only responses)
- **`Unhandled`**: Route with polymorphic errors requiring handling
- **`Provided`**: Route with environment already provided
- **`Augmented`**: Route with handler transformation applied
