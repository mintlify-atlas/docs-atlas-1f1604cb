ZIO HTTP provides a powerful declarative endpoint API that allows you to define HTTP endpoints with compile-time type safety and automatic OpenAPI documentation generation.

## Overview

Declarative endpoints separate the endpoint description from its implementation, providing several benefits:

- **Type Safety**: Input and output types are checked at compile time
- **OpenAPI Generation**: Automatic OpenAPI/Swagger documentation
- **Code Generation**: Generate client code from endpoint definitions
- **Documentation**: Inline documentation with the `??` operator
- **Validation**: Built-in validation with meaningful error messages

## Creating Endpoints

Define an endpoint using the `Endpoint` constructor:

```scala
import zio.http._
import zio.http.endpoint._
import zio.http.codec._

val getUserEndpoint = 
  Endpoint(Method.GET / "users" / int("id"))
    .in[Unit]
    .out[User]
```

## Input Specification

Specify endpoint inputs using `.in`, `.query`, and `.header`:

```scala
val createUserEndpoint = 
  Endpoint(Method.POST / "users")
    .in[CreateUserRequest]
    .header(HeaderCodec.authorization)
    .out[User]
```

## Query Parameters

```scala
val searchEndpoint = 
  Endpoint(Method.GET / "search")
    .in[Unit]
    .query(QueryCodec.queryStr("q"))
    .query(QueryCodec.queryInt("limit").optional)
    .out[SearchResults]
```

## Implementation

Implement an endpoint with a handler:

```scala
val getUserRoute = 
  getUserEndpoint.implement {
    handler { (userId: Int) =>
      UserService.getById(userId)
    }
  }
```
