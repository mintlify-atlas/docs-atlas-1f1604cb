This example demonstrates how to build a WebSocket server using ZIO HTTP. You'll learn how to handle WebSocket connections, process incoming messages, send messages to clients, and manage connection lifecycle events.

## Complete Example

Here's a complete WebSocket server with echo functionality and advanced features:

```scala
package example.websocket

import zio._
import zio.http.ChannelEvent.{ExceptionCaught, Read, UserEvent, UserEventTriggered}
import zio.http._
import zio.http.codec.PathCodec.string

object WebSocketChat extends ZIOAppDefault {

  val socketApp: WebSocketApp[Any] =
    Handler.webSocket { channel =>
      channel.receiveAll {
        case Read(WebSocketFrame.Text("end")) =>
          channel.shutdown

        // Send a "bar" if the client sends a "foo"
        case Read(WebSocketFrame.Text("foo")) =>
          channel.send(Read(WebSocketFrame.text("bar")))

        // Send a "foo" if the client sends a "bar"
        case Read(WebSocketFrame.Text("bar")) =>
          channel.send(Read(WebSocketFrame.text("foo")))

        // Echo the same message 10 times if it's not "foo" or "bar"
        case Read(WebSocketFrame.Text(text)) =>
          channel
            .send(Read(WebSocketFrame.text(s"echo $text")))
            .repeatN(10)
            .catchSomeCause { case cause =>
              ZIO.logErrorCause(s"failed sending", cause)
            }

        // Send a "greeting" message to the client once the connection is established
        case UserEventTriggered(UserEvent.HandshakeComplete) =>
          channel.send(Read(WebSocketFrame.text("Greetings!")))

        // Log when the channel is getting closed
        case Read(WebSocketFrame.Close(status, reason)) =>
          Console.printLine("Closing channel with status: " + status + " and reason: " + reason)

        // Print the exception if it's not a normal close
        case ExceptionCaught(cause) =>
          Console.printLine(s"Channel error!: ${cause.getMessage}")

        case _ =>
          ZIO.unit
      }
    }

  val routes: Routes[Any, Response] =
    Routes(
      Method.GET / "greet" / string("name") -> handler { (name: String, _: Request) =>
        Response.text(s"Greetings ${name}!")
      },
      Method.GET / "subscriptions" -> handler(socketApp.toResponse),
    )

  override val run = Server.serve(routes).provide(Server.default)
}
```

## What This Example Demonstrates

This WebSocket example showcases several important real-time communication patterns:

- **WebSocket Handshake**: Upgrading HTTP connections to WebSocket
- **Bidirectional Communication**: Sending and receiving messages
- **Message Pattern Matching**: Handling different message types
- **Connection Lifecycle**: Managing handshake, messages, and close events
- **Error Handling**: Gracefully handling exceptions
- **Message Broadcasting**: Sending multiple messages (echo pattern)

### Process Text Messages

Handle different text messages with pattern matching:

```scala
case Read(WebSocketFrame.Text("foo")) =>
  channel.send(Read(WebSocketFrame.text("bar")))

case Read(WebSocketFrame.Text("bar")) =>
  channel.send(Read(WebSocketFrame.text("foo")))
```

This creates a simple "ping-pong" pattern where specific messages get specific responses.

For unmatched messages, echo them back multiple times:


- `.repeatN(10)` sends the message 11 times total (original + 10 repeats)
- `.catchSomeCause` handles any errors that occur during sending

Gracefully handle when clients disconnect:

```scala
case Read(WebSocketFrame.Close(status, reason)) =>
  Console.printLine("Closing channel with status: " + status + " and reason: " + reason)

case Read(WebSocketFrame.Text("end")) =>
```

The server can close connections programmatically or respond to client close frames.

Catch and log exceptions:


## How to Run

Save the code and run it:

```bash
scala-cli run WebSocketChat.scala
```

The server starts on port 8080 with WebSocket support.

### Using JavaScript in Browser

Create an HTML file to test from the browser:

```html
<!DOCTYPE html>
<html>
<head>
  <title>WebSocket Test</title>
</head>
<body>
  <h1>WebSocket Chat</h1>
  <div id="messages"></div>
  <input type="text" id="messageInput" placeholder="Type a message">
  <button onclick="send()">Send</button>

  <script>
    const ws = new WebSocket('ws://localhost:8080/subscriptions');
    const messages = document.getElementById('messages');

    ws.onopen = () => console.log('Connected!');
    
    ws.onmessage = (event) => {
      const p = document.createElement('p');
      p.textContent = 'Server: ' + event.data;
      messages.appendChild(p);
    };

    ws.onerror = (error) => console.error('WebSocket error:', error);
    
    ws.onclose = () => console.log('Disconnected');

    function send() {
      const input = document.getElementById('messageInput');
      ws.send(input.value);
      const p = document.createElement('p');
      p.textContent = 'You: ' + input.value;
      messages.appendChild(p);
      input.value = '';
    }
  </script>
</body>
</html>
```

## Building a Chat Room

Here's how to extend this into a multi-user chat room:

```scala
object ChatRoom extends ZIOAppDefault {
  // Store all connected clients
  case class ChatState(clients: Set[WebSocketChannel])
  
  val chatRoom = ZIO.serviceWithZIO[Ref[ChatState]] { state =>
    Handler.webSocket { channel =>
      for {
        // Add this client to the room
        _ <- state.update(s => s.copy(clients = s.clients + channel))
        
        // Handle messages
        _ <- channel.receiveAll {
          case Read(WebSocketFrame.Text(text)) =>
            // Broadcast to all clients
            state.get.flatMap { chatState =>
              ZIO.foreachDiscard(chatState.clients) { client =>
                client.send(Read(WebSocketFrame.text(text)))
                  .catchAll(_ => ZIO.unit) // Ignore errors for dead connections
              }
            }
          
          case Read(WebSocketFrame.Close(_, _)) =>
            // Remove this client from the room
            state.update(s => s.copy(clients = s.clients - channel))
          
          case _ => ZIO.unit
        }
      } yield ()
    }
  }

  val routes = Routes(
    Method.GET / "chat" -> handler(chatRoom.toResponse)
  )

  override val run = 
    Server.serve(routes)
      .provide(
        Server.default,
        ZLayer.fromZIO(Ref.make(ChatState(Set.empty)))
      )
}
```

## Key Takeaways

- **Event-Driven**: WebSocket communication is based on handling different event types
- **Bidirectional**: Both server and client can initiate message sending
- **Pattern Matching**: Use Scala's pattern matching to handle different message types elegantly
- **Lifecycle Management**: Handle handshake, messages, errors, and close events
- **Backpressure**: ZIO HTTP handles backpressure automatically
- **Type Safety**: Frame types (Text, Binary, Close, Ping, Pong) are type-checked
