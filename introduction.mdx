```scala
import zio._
import zio.http._

object GreetingServer extends ZIOAppDefault {
  val routes =
    Routes(
      Method.GET / Root -> handler(Response.text("Greetings at your service")),
      Method.GET / "greet" -> handler { (req: Request) =>
        val name = req.queryOrElse[String]("name", "World")
        Response.text(s"Hello $name!")
      }
    )

  def run = Server.serve(routes).provide(Server.default)
}
```

## Why ZIO HTTP?

ZIO HTTP offers several advantages for building HTTP applications:

<Accordion title="Type Safety">
  ZIO HTTP leverages Scala's type system to ensure correctness at compile time. Route patterns, request parameters, and response types are all type-checked, catching errors before runtime.
</Accordion>

<Accordion title="Composability">
  Built on ZIO, ZIO HTTP inherits powerful composition patterns. Routes, handlers, and middleware can be easily combined and reused across your application.
</Accordion>

<Accordion title="Performance">
  Powered by Netty, ZIO HTTP delivers high throughput and low latency. Built-in support for streaming and backpressure ensures efficient resource utilization.
</Accordion>

<Accordion title="Developer Experience">
  From declarative endpoint definitions to automatic OpenAPI generation, ZIO HTTP provides excellent tooling and IDE support for productive development.
</Accordion>

## What's Included

ZIO HTTP provides everything you need to build production-ready HTTP applications:

- **Server & Client**: Full-featured HTTP server and client implementations
- **Routing**: Flexible routing DSL with path parameter extraction
- **Middleware**: Cross-cutting concerns like logging, authentication, and CORS
- **WebSockets**: First-class WebSocket support for real-time communication
- **Streaming**: Request and response body streaming with ZIO Streams
- **SSL/TLS**: Comprehensive SSL/TLS configuration and certificate management
- **OpenAPI**: Automatic OpenAPI documentation generation
- **Testing**: TestKit for testing HTTP applications without a live server
- **Templates**: Built-in DSL for server-side HTML rendering
- **Codecs**: Integration with ZIO Schema for JSON, Protobuf, Avro, and Thrift
